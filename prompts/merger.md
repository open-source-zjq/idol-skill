# 增量合并逻辑

当用户提供新的微博数据时，需要将新内容与已有数据合并。

## 合并规则

### 风格语料（style_corpus）
1. 按 id 去重：新语料中与已有 id 重复的条目跳过
2. 新增条目追加到语料末尾
3. 合并后需要重新运行风格分析

### 风格画像（style.md）
1. 读取现有 style.md
2. 用新的风格分析结果与现有内容对比
3. 分类处理：
   - **补充型**：新发现的特征 → 追加到对应层级
   - **确认型**：与现有描述一致 → 不变
   - **矛盾型**：与现有描述冲突 → 标记给用户确认
4. 不要删除已有内容，只追加或标记

## 输出

完成合并后：
1. 更新 data/ 下的 JSON 文件
2. 重新生成 style.md
3. 重新生成 SKILL.md
4. 版本号递增
